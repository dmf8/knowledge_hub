# static
特点
1. 所有的对象和类共享
1. 可以通过类和对象访问
1. 存在权限

## 静态成员变量
使用要点:
1. 编译时分配空间
1. 类外初始化, 因为类内的部分需要在构造对象时才会执行

    特例: static const int可以在类内初始化
    ```cpp
    static const int i2 = 2; //除此之外, 静态变量都需要在类的cpp文件中进行类外初始化
    ```

1. 不要在头文件进行类外初始化, 当头文件被多次引用时会报重定义的错(多个强符号)


## 静态成员函数
使用要点:
1. 和静态成员函数不同, 在类内初始化(直接在头文件中写出实现)或类外初始化均可

    可以直接在.h中定义的原因是会被隐式加上inline

    但一般都把实现写在.cpp中, 因此不会使用类内初始化的方式

# 成员空间
## 空间实例数量
1. 非静态变量在每个对象中分开存储
1. 静态变量只占一份空间
1. 非内联函数只有一份函数实例

## 对象大小
1. 空类大小为1

    空类的对象也需要维护一个地址, 需要一个char来占用这个地址

1. 一般对象的大小由非静态成员变量决定, 需要兼顾字节对齐

# this
这个指针指向当前对象本身, 不占用对象空间

使用场景:
1. 链式编程, 成员函数声明为返回`T&`, 在函数实现中返回`*this`
1. 和形参或局部变量冲突时, 区分是否是成员变量(根据就近原则, 不带this的是局部变量)
1. 成员函数内部没有用到this, 可以用空指针调用该函数

# const/mutable
## 常函数
带有const后置修饰的成员函数, 特点为:
1. 函数内部的this带有const修饰, 通常不能修改对象的值
1. 如果成员变量带有mutable修饰, 不受常函数约束

## 常对象
const前置修饰的对象, 特点为:
1. 只能调用常函数
1. 仍然可以修改mutable修饰的成员变量

# friend
友元, 定义在类的外部, 但可以访问类的protected/private成员.
